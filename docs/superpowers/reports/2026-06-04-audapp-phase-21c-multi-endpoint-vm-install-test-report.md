| Endpoint GUID | SWD FriendlyName | State |

### Endpoint Visibility Notes

- **Four render circuits confirmed**, each mapping to a distinct KS circuit (speakergeneral / speakermusic / speakervoice / speakergame). ✓
- **Per-circuit INF FriendlyNames** ("Audapp General", "Audapp Music", etc.) are **not propagated** to the MMDevice `PKEY_Device_FriendlyName` (`{a45c254e...},14`). This is expected behavior for ACX 1.0 on Windows 10: the AudioEndpointBuilder generates a generic "Hoparlör (Audapp Multi)" name for all circuits on the same device.
- **KS circuit identity** IS embedded in the endpoint's `{233164c8...},1` property (e.g., `\\?\root#devgen#audappmulti21c0001#{6994ad04...}\speakervoice`), which is accessible to code querying `IMMDeviceCollection`.
- **Phase 21D design item**: implement per-circuit FriendlyName (via KSPROPERTY_PIN_NAME on the bridge pin, or equivalent ACX DDI) so Windows-facing endpoint names are "Audapp General" etc. rather than "Hoparlör (Audapp Multi)".

**QUALIFIED PASS** — Four endpoints are visible and circuit-distinct. FriendlyName gap is a Phase 21D item, not a blocker.
