# Audapp Input driver package staging



Compile-only builds stage install-critical artifacts here. Phase 12D adds catalog generation via `Generate-Catalog.ps1`. Phase 12E adds VM-only catalog signing via `Sign-Catalog.ps1` (elevated; no install).



## Layout



```text

package\<Configuration>\<Platform>\

  AudioCodec.sys   # binary name follows upstream ACX sample output

  AudioCodec.inf   # stamped INF with Audapp Input package-facing strings

  audiocodec.cat   # generated by Generate-Catalog.ps1; signed by Sign-Catalog.ps1

  package-manifest.txt

```



Default path after `build.ps1`:



```text

package\Debug\x64\

```



Generate the catalog (from `driver\scaffold\audapp-input`):



```powershell

powershell -ExecutionPolicy Bypass -File .\Generate-Catalog.ps1

```



Uses WDK `Inf2Cat.exe` against the staged folder (default OS target: `10_VB_X64`).



Sign the catalog (elevated VM PowerShell only):



```powershell

powershell -ExecutionPolicy Bypass -File .\Sign-Catalog.ps1

powershell -ExecutionPolicy Bypass -File .\Sign-Catalog.ps1 -SignSys   # optional SYS signing

```



Public certificate export defaults to `%USERPROFILE%\Documents\Audapp\driver-test-signing\`.



## Out of scope here



- driver install, load, or test-signing boot changes (`bcdedit`)

- `pnputil` / `devcon`
